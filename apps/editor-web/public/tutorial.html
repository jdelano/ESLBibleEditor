KJVeasy-ISL Editor Tutorial

This guide walks through the whole alpha workflow: choosing a verse, adding token and group annotations, making links, using colors and part-of-speech tagging, working with margin notes, importing and exporting JSON, and fixing validation issues.

Quick start

  1. Open the editor and choose a Book, Chapter, and Verse in the Bible Browser.
  2. If the verse has been edited before, the saved local version opens. If not, the KJV text is loaded and auto-tokenized.
  3. Click a token or annotation lane to select where you want to work.
  4. Use the toolbar to add annotations, create groups, add margin notes, or make links.
  5. Click Save Verse when you want to persist the verse in browser storage.
  6. Use Export JSON to download the current verse, or the toolbar export/publish tools for broader packaging.

Layout overview

Left pane

Bible Browser. Choose the book, chapter, and verse you want to edit.

Center pane

Verse canvas. This is where tokens, groups, annotations, links, and margin notes are shown and edited.

Right pane

Inspector. Shows fields for the currently selected token, annotation, group, link, media item, or slot.

Bottom pane

Diagnostics. Validation issues appear here with highlight and repair actions.

Choose a verse

Use the Bible Browser dropdowns to navigate by book, chapter, and verse. The app loads one verse at a time.

How loading works: if a verse already exists in local browser storage, the saved version opens. If it does not, the editor pulls the KJV text from kjv.json, tokenizes it, assigns part-of-speech tags, and applies word colors automatically.

Work with tokens

Each word is shown as a token card. Tokens are the basic editing unit.

  • Click a token to toggle it on or off.
  • Click multiple tokens in sequence to build a multi-selection.
  • If you click an annotation or lane after selecting tokens, the token selection clears and the new item is selected instead.
  • When multiple tokens are selected, the inspector lets you apply shared edits like POS, color, proper noun, and editorial flags.

Add and edit annotations

An annotation is edited directly in its bubble on the canvas. You are not limited to text only. A single annotation can mix text, emoji, and pasted images on one line.

  • Click an empty annotation lane to select that slot.
  • Use Add Ann in the toolbar to create a new annotation in that exact spot.
  • Click the annotation bubble to edit it inline.
  • Type text directly into the bubble.
  • Paste emoji directly into the text while editing.
  • Paste an image from the clipboard to insert a cropped inline image segment.
Editing keys: Enter finishes editing. Arrow keys move between text and inline images. Backspace and delete work across text and image boundaries.

Groups and grouped lanes

Token groups let several contiguous tokens behave like one visual unit while each underlying token still keeps its own identity.

  • Select contiguous tokens and click Group to make a token group.
  • Click Ungroup to remove the token group.
  • Inside a group, each annotation row can be shared across the group or split per token.
  • You can also create grouped annotation subranges inside a token group by selecting contiguous empty or occupied lanes in the same row and pressing Group.
  • Split Lane converts a grouped lane back into split per-token slots.
Important: token grouping and annotation grouping are related but not the same. A token group can exist even if an annotation row is currently split.

Margin notes

Each token group has a dedicated top margin lane. That lane always spans the whole group and is used for the group’s margin note.

  • Select a group and click Margin to create a margin note.
  • Click the margin note lane to edit it inline.
  • The inspector lets you override the margin note color.
  • Margin notes can contain text, emoji, and pasted images just like regular annotations.

Pronoun and antecedent links

Links are drawn as routed arrows below the token cards. They are commonly used for pronouns and antecedents, but you can use them for any token-to-token relationship you need to show.

  • Drag from one token to another to create a directed link.
  • The drag direction determines the arrow direction.
  • Select a link to edit its label, type, or color in the inspector.
  • Use Link Left or Link Right in the toolbar to flip the selected arrow direction.

POS tags and word colors

Tokens can be annotated linguistically using both part-of-speech and semantic color categories.

  • Select a token and use the inspector to set Part of Speech.
  • Select a token and use the inspector to set Word Color.
  • Use the toolbar POS button to auto-tag the current verse.
  • Use the toolbar Colors button to auto-assign word colors.
Word color examples: God, good angels, bad angels, person name, group name, people, things, place name, place, time, and number.

Paste and edit images inside annotations

Images are stored inline as base64 segments and are cropped to a small annotation-friendly square.

  • Select an annotation and paste an image from the clipboard.
  • Images can appear between text segments, not only at the end.
  • Click an image segment to focus it.
  • Use arrow keys to navigate across text and image segments.
  • Use backspace or delete to remove an image.
  • Drag an existing inline image to another position in the same annotation to reinsert it as an image segment.

Validation and repair

The diagnostics pane beneath the center canvas shows validation errors and warnings for the current verse.

  • Each issue has a Highlight button to jump to the related verse object.
  • Some issues also have a direct repair action.
  • Repair Verse performs safe cleanup for common stale-reference problems.
Typical repair cases include orphaned annotations, broken links, stale layout references, and media-order cleanup.

Import, export, and save

  • Save Verse writes the current verse into browser local storage.
  • Import JSON replaces the current verse with a verse JSON file.
  • Export JSON downloads the current verse JSON.
  • Chapter/book export buttons create larger package files based on the local browser data.
Single-user model: this app is browser-first. Data is stored locally in your browser unless you export it. If you open the app on a different machine or browser, you will need to import your JSON there.

Practical tips

  • Use groups whenever several tokens should share a common annotation lane or margin note.
  • Use grouped lanes for shared phrases; use split lanes when each token needs its own annotation in the same row.
  • Keep links for relationships under the text and annotations for explanatory content above the text.
  • If layout starts feeling messy, check diagnostics before exporting or publishing.
  • Save often, especially after major grouping or annotation changes.
